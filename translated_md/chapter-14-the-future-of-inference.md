# 第 14 章：推理的未来

_来源页：PDF 第 50-52 页_

## 14.1 基于扩散的语言模型

自回归生成，即一次生成一个 token，并不是唯一范式。扩散语言模型（MDLM、Plaid、LLaDA）将扩散过程应用于离散 token 序列：前向过程逐步遮蔽 token，模型则学习对其去噪（解除遮蔽）。与自回归模型不同，扩散 LLM 可以并行（或半并行）生成所有 token，因此在生成期间有望实现高得多的硬件利用率。

挑战在于：用于文本的扩散模型在质量上尚未达到自回归模型的水平，而且迭代式去噪过程（通常为 10-100 个去噪步骤，每一步都会触及所有 token）可能比预期更昂贵。不过，LLaDA 等近期模型已经显著缩小了差距。

## 14.2 多 token 预测

Meta 的多 token 预测（Gloeckle et al., 2024）使用多个输出头训练模型，使其同时预测接下来的 \(k\) 个 token，而不只是一个 token。在推理期间，这可以像 Medusa（自投机解码）一样用于每一步生成多个 token；也可以利用多个输出头为下游任务生成更好的内部表征。采用 4-token 预测训练的模型，相比标准的 next-token prediction，表现出更好的样本效率和表征质量。

## 14.3 神经形态与光子计算

神经形态芯片（Intel Loihi 2、IBM NorthPole）实现了类似神经元的脉冲计算，对稀疏激活极为节能：按每次操作计算，其能效比 GPU 深度学习高出数个数量级。IBM 的 NorthPole 芯片在密集神经推理中达到 22 TOPS/watt，而 GPU 约为 1 TOPS/watt。随着 LLM 架构向稀疏性演进，神经形态硬件可能会变得相关。

光子矩阵乘法器（Lightmatter、Luminous Computing）使用光来执行矩阵-向量乘法，理论上可实现无限带宽，并且每次乘法的能耗接近于零。光子计算仍处于早期阶段，但它可能从根本上改变内存带宽瓶颈，因为它可以让权重加载几乎免费。

## 14.4 推理优先的设计理念

该领域正日益转向从一开始就以推理为中心来设计模型：

- **面向 KV 效率的架构选择**：MQA、GQA、MLA 和 SSM 层可减少 KV cache 占用。
- **激活稀疏性**：使用 ReLU 或 top-K 激活函数训练的模型具有稀疏激活，可用于加速 FFN 计算。
- **Mixture of Depths**（Raposo et al., 2024）：一种动态计算方法，其中 token 会根据难度只经过较少的层。
- **Early exit**：对于容易的 token，在中间层基于置信度提前停止。
- **字节级模型**：在字节而不是子词 token 上操作，可以消除 tokenization 瓶颈，并实现更好的多语言性能。

## 面试准备：模拟问答

### Q14.1 扩散语言模型生成文本的方式与自回归模型有何不同？这对推理有什么影响？

答案：自回归模型通过一次预测一个 token 来生成文本，并以所有先前 token 为条件；这在本质上是顺序的，并且受内存带宽限制。扩散语言模型则同时作用于整个序列：从目标长度的全遮蔽序列开始，然后在 \(T\) 个步骤中使用模型预测应解除遮蔽的 token，并迭代去噪（解除遮蔽）。这带来几个关键推理影响：（1）**并行性**：每个去噪步骤同时触及所有位置，与自回归 decode 不同，可以充分利用 tensor cores；（2）**可控性**：扩散模型可以自然地填充被遮蔽部分（infilling/editing），无需从头重新生成；（3）**成本结构**：每次生成需要 \(T \times N\) 次 token 预测，而自回归只需要 \(N\) 次，因此在相同输出长度下可能更昂贵；（4）**长度灵活性**：需要预先知道输出长度（或将其作为超参数处理）。目前 LLaDA 等模型使用 \(T = 64\) 到 \(256\) 个步骤，不过蒸馏可以将其降至 4-8 个步骤。

### Q14.2 什么是 Mixture of Depths？它如何实现自适应推理计算？

答案：Mixture of Depths（Raposo et al., 2024）是一种框架，其中并非所有 token 都会通过所有 transformer 层。每一层的路由器决定某个 token 是否应由该层的 attention 和 FFN 处理，或者只是通过 residual connection 直接传递（跳过该层）。容易/可预测的 token 会跳过层，而复杂/不确定的 token 则会获得完整处理。在推理时，这提供了自适应计算：简单事实查询可能实际上只使用模型 30% 的层，而复杂推理查询会使用所有层。这不同于 early exit（它在固定层停止生成）：MoD 会让被跳过的 token 继续进入后续层。训练需要谨慎的路由，通常是在每层按 token 使用 top-K selection。对于许多 token 分布，MoD 可以在 50% FLOP 成本下达到与密集模型相近的 perplexity。

### Q14.3 为什么光子计算可能从根本上改变 LLM 推理？当前有哪些限制？

答案：光子矩阵乘法利用光波干涉以光速计算点积，并且每次乘法的能耗接近于零（光子不会加热电阻）。它对 LLM 推理的理论收益包括：（1）**带宽墙消失**：光互连可以在单根光纤上传输每秒数太比特数据，远超 HBM 的限制；（2）**能效**：相比 GPU CMOS，每 FLOP 能耗可降低 100-1000 倍；（3）**延迟**：光速传播消除了电子开关延迟。当前限制包括：（1）**数字到光的转换**：将权重和激活加载为光学形式仍需要电子 ADC/DAC，而它们速度慢且耗电；（2）**精度**：当前光子系统实现的是 4-8 bit 等效精度，足以用于推理，但需要量化；（3）**可编程性**：重新编程光学矩阵元件比在 GPU 上加载新权重更慢；（4）**可扩展性**：当前光子芯片实现的是小规模矩阵乘法（\(64 \times 64\) 到 \(512 \times 512\)），远小于 LLM 权重矩阵。截至 2025 年，Lightmatter 的 Passage 芯片是最先进的商用光子推理加速器。

### Q14.4 多 token 预测如何同时提升训练和推理效率？

答案：多 token 预测使用 \(k\) 个预测头训练模型，每个头都向未来多预测一步：head 1 预测 \(t + 1\)，head 2 预测 \(t + 2\)，依此类推。所有头共享同一个主体表征。训练收益包括：（1）每次 forward pass 提供更多梯度信号，模型同时优化 \(k\) 个预测目标；（2）更好的长程表征，因为要预测 \(t + 4\)，模型必须捕捉超越局部共现的更高层模式；（3）Meta 的实验表明，3-4 token 预测可以提升基准性能和编码能力，尤其是在较小模型上。推理收益包括：额外预测头可以在不增加模型尺寸的情况下实现自投机解码：使用 head 2 到 head \(k\) 起草 token \(t + 2\) 到 \(t + k\)，然后用下一次主 forward pass 验证。这提供了 Medusa 风格的加速，而不需要单独的 draft model。接受率中等（每个头约 0.7-0.8），在编码任务上可带来 1.8-2.5 倍 decode 加速。

### Q14.5 描述一种“推理优先”的模型设计理念。哪些架构选择体现了这种方法？

答案：推理优先的理念从一个问题出发：“生产部署会施加哪些约束？”然后反向推导架构决策，而不是在训练时最大化基准分数，之后再尝试事后优化。体现推理优先思想的关键架构选择包括：（1）**GQA/MQA/MLA**：更少的 KV heads 会显著减少 KV cache 内存，直接支持更大的 batch size 和更长的上下文；（2）**SwiGLU FFN**：门控架构允许在 MoE 变体中基于 routing scores 剪枝整个专家，从而实现条件计算；（3）**激活稀疏性（ReLU 变体）**：稀疏激活模式（Mistral 声称使用 ReLU 可达到 89% FFN 稀疏性）支持跳过 FFN 计算的大部分；（4）**缩小词表**：更小的词表意味着更小的 embedding tables 和更快的采样；（5）**字节级或字符级 tokenization**：消除 tokenization 瓶颈，改善多语言处理；（6）**深度优先于宽度**：更窄、更深的模型可能更难训练，但可以更高效地在 GPU 上并行。Phi-3 和 Gemma 体现了推理优先理念：它们是有意设计的小模型，优化的是每个参数带来的质量，而不是绝对基准最大化。
