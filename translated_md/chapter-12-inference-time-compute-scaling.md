# 第 12 章：推理时计算扩展

## 12.1 新的扩展范式

多年来，主导性的扩展定律是：更多训练计算量 -> 更好的模型。Chinchilla 最优训练建议将计算预算大致平均分配给模型规模和训练 token。但在 2024-2025 年，一个互补范式开始兴起：推理时计算扩展，即在推理时投入更多计算量来提升输出质量。

关键洞见是：对许多任务而言，尤其是推理、数学和代码任务，一个“思考更久”的较小模型可以超过一个“立即回答”的较大模型。问题从“我们能训练多大的模型？”转变为“我们在推理时负担得起多少推理计算？”

## 12.2 Chain-of-Thought 及其推理成本

Chain-of-Thought (CoT) 提示（Wei et al., 2022）要求模型在给出最终答案之前生成中间推理步骤。这会显著提升多步推理任务的性能，但也带来成本：一个只需要 10 个 token 答案的问题，可能需要 500 个 token 的推理过程，使推理成本扩大 50×。

**推理时计算扩展问题：**在给定固定计算预算的情况下，是运行一次更大的模型更好，还是使用更小的模型并让它生成更多推理 token 更好？

近期结果（Snell et al., 2024，“Scaling LLM Test-Time Compute Optimally”）表明，答案取决于问题难度：对于困难问题，在较小模型上投入更多推理计算通常更高效；对于简单问题，扩展推理带来的开销则是一种浪费。

## 12.3 Process Reward Models 与搜索

Process Reward Model (PRM) 被训练用于评估中间推理步骤的质量，而不仅仅是最终答案。PRM 使搜索成为可能：不再只沿着单条推理链前进，而是生成多个候选推理路径，并用 PRM 选择或重排最有希望的路径。

常见搜索策略包括：

- **Best-of-N**：生成 \(N\) 个独立解答，并选择 PRM 分数最高的一个，或进行多数投票。
- **Beam search**：维护 \(K\) 条部分推理路径，并在每一步扩展最有希望的路径。
- **Monte Carlo Tree Search (MCTS)**：在树搜索中将 PRM 用作价值函数。

在数学基准（MATH、AIME）上，\(N = 64\) 的 Best-of-N 可以达到或超过大 10× 的模型性能。

## 12.4 DeepSeek-R1 与 o1 风格架构

OpenAI o1 和 DeepSeek-R1 代表了一种质变：这些模型不是通过提示来产生 CoT，而是通过强化学习训练出推理能力。关键机制是：模型在给出最终答案之前，会生成扩展的内部独白（“thoughts”）。这些 thoughts 通过 RLHF/GRPO，以可验证奖励作为目标进行训练，例如数学答案的正确性：

\[
L_{\mathrm{GRPO}} =
\mathbb{E}_{o}\left[
\min\left(
\frac{\pi_\theta(o\mid q)}{\pi_{\mathrm{ref}}(o\mid q)} A_o,
\operatorname{clip}(\cdot, 1-\epsilon, 1+\epsilon) A_o
\right)
\right]
+ \beta D_{\mathrm{KL}}(\pi_\theta \| \pi_{\mathrm{ref}})
\]

结果是，模型能够在困难问题上“思考”数千个 token，用推理成本换取准确率。DeepSeek-R1-Zero 表明，这种推理行为可以在没有监督式推理轨迹的情况下，仅通过纯 RL 涌现出来。

## 面试准备：模拟问答

### Q12.1 什么是推理时计算扩展？它与训练时计算扩展有何不同？

**答案：**训练时计算扩展（Kaplan et al., 2020；Hoffmann et al., 2022 - Chinchilla）研究模型能力如何随训练 FLOP 预算增长而提升，即更多参数和更多训练 token 会产生更好的模型。推理时计算扩展是一个较新的范式：在给定一个已经训练好的固定模型时，如果我们在推理时投入更多计算量，性能会如何扩展？更多推理计算可以通过以下方式投入：（1）生成更多推理 token（Chain-of-Thought）；（2）生成多个候选解并选择最佳解（Best-of-N sampling）；（3）使用 verifier 或 reward model 执行树搜索；（4）迭代式优化，即模型批判并改进自己的输出。关键洞见是，推理计算可以替代训练计算：在某些推理任务上，一个投入 64× 更多推理计算的 7B 模型可以匹配一个 70B 模型。这改变了 AI 部署的经济学：计算预算从一次性的训练成本，转移为按查询产生的推理成本。

### Q12.2 解释 Process Reward Models (PRMs)。它们与 Outcome Reward Models (ORMs) 有何不同？

**答案：**Outcome Reward Model (ORM) 会为完整解答分配单个奖励分数，并且只基于最终答案是否正确。它无法区分一个通过错误推理得到正确答案的解答，和一个通过有效推理得到正确答案的解答。ORM 也无法在中间步骤上指导搜索。Process Reward Model (PRM) 被训练为每个单独的推理步骤打分，以判断其正确性，从而在整条推理链中提供密集反馈。PRM 需要训练数据中的步骤级标注，这类标注采集成本很高，尽管也存在自动化方法。在推理时，PRM 能够在步骤级别启用搜索：MCTS、beam search 和步骤级重排序都变得可行。对于复杂的多步推理，尤其是中间路径很重要的场景，PRM 优于 ORM，因为它能防止模型“用错误理由得到正确答案”；但 PRM 的训练和应用成本也显著更高。

### Q12.3 什么是 Best-of-N sampling？它什么时候高效，什么时候不高效？

**答案：**Best-of-N (BoN) sampling 会从模型中生成 \(N\) 个独立候选输出（temperature > 0），用 reward model 或 verifier 为每个候选打分，并返回分数最高的输出。它是最简单的推理时计算扩展方法之一。**高效场景：**（1）具有可验证正确答案的任务，例如数学、代码和形式逻辑，此时 verifier 便宜且可靠；（2）生成结果多样性较高时，如果所有 \(N\) 个样本都相同，BoN 就没有收益；（3）\(N\) 较小（\(\le 32\)）且单样本推理成本较低时。**低效场景：**（1）没有可靠评估器的开放式任务，例如创意写作，因为主观质量很难自动评分；（2）非常大的 \(N\)，在许多设置中 BoN 质量按 \(O(\log N)\) 增长，也就是从一个分布中取 \(N\) 个样本的期望最大值，收益递减明显；（3）模型很昂贵的场景，因为 \(N\) 个样本的成本比单个样本高 \(N\times\)，在相同计算预算下，使用 beam search 或步骤级优化往往更好。

### Q12.4 描述 DeepSeek-R1 中使用的 GRPO 训练目标。它与 PPO 有何不同？

**答案：**GRPO (Group Relative Policy Optimization) 是一种用于 LLM 微调的 RL 算法，它避免了对独立 value network（critic）的需求。PPO 使用 value network 估计 advantage function \(A\)，而 GRPO 则根据同一个问题生成的一组输出，计算每个输出相对于该组的 advantage：\(A_i = (r_i - \mathrm{mean}(r)) / \mathrm{std}(r)\)，其中 \(r\) 是该组输出的奖励向量。这消除了训练 value function 的需求，节省内存和计算量；通过组内归一化降低方差；并避免了 value function 估计的不稳定问题。与 PPO 相比，GRPO 更简单，超参数更少；内存效率更高，因为不需要 value network；但由于 baseline 是从当前 batch 估计出来的，每次更新的方差可能更高。DeepSeek-R1-Zero 表明，仅靠 GRPO 加简单的可验证奖励，就能够诱导出自我验证和扩展思考等复杂推理行为。

### Q12.5 对于数学辅导应用，你会使用更大的基础模型，还是使用带扩展推理计算的较小模型？请说明理由。

**答案：**我会选择带扩展推理计算的较小模型，理由有几个：（1）数学是可验证的：正确答案可以通过程序检查，从而无需人工评估也能可靠地使用 Best-of-N 或 PRM 引导搜索；（2）推理轨迹具有教学价值：一个展示逐步解题过程的数学导师，比只给答案的导师更有用；推理时的 CoT 是功能，而不是额外负担；（3）成本效率：即使计入额外 token，服务一个带扩展推理的 7B R1 风格模型，每次查询的成本也低于服务一个 70B 稠密模型；（4）计算预算可自适应：对于简单问题，可以使用较少推理 token；对于困难问题，可以使用更多推理 token，从而提供计算自适应的服务能力，这是固定的大模型无法做到的。实际选择会是类似 7B 或 14B 的微调推理模型，例如 DeepSeek-R1-Distill 或类似模型，并配备可配置的推理预算，而不是一个 70B 基线模型。
